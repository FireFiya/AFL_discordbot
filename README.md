# Discord 請假機器人

一個用於 Discord 伺服器的請假管理機器人。員工透過**按鈕面板**申請、查看、取消請假，管理員一次設置面板後即可長期使用。<br/>
此機器人是為解決discord內請假日期過難統整與管理而設計

## 功能特色

- 🏢 **持久按鈕面板** — 設置一次即可，機器人重啟後仍有效，員工全程用按鈕操作，不需打指令
- 📅 **請假申請** — 階層式選單（年 → 月 → 上/下半月 → 日），自動擋掉過去的日期，並防止重複請假
- 🔁 **定期請假** — 可設定「每周X」，自動排入當月所有該星期幾；每月 1 號自動續排下個月
- 📋 **月曆檢視** — 以圖片呈現當月每天的請假**人數**（不顯示姓名以保護隱私），可切換月份；下拉選單可查看某天「是誰」請假
- ✖️ **取消請假** — 員工可取消自己的請假，定期請假可整組取消
- 📢 **自動提醒**
  - 有人申請**當天**請假時，即時公開提醒大家
  - 有人取消**當天**請假時，公開提醒並更新今日請假人數
  - 每天早上（台灣時間 09:00）若今天有人請假，自動到面板頻道公告名單
  - 每月 1 號產生定期請假後，於頻道發一則總結（不 @ 任何人）
- 🗄️ **歷史封存** — 過期的請假不會被刪除，而是搬到封存檔保存，月曆可往回查看歷史

## 指令

| 指令 | 說明 | 權限 |
|------|------|------|
| `!設置面板` | 發送請假面板（詳見下方權限規則） | 依身分不同 |
| `!清空請假` | 清空目前（今天與未來）的請假記錄，歷史封存保留 | 需「管理伺服器」權限 |

### `!設置面板` 的頻道白名單機制

為了避免面板被隨意張貼到其他頻道，採用「指定頻道」機制：

- **管理員**（具「管理伺服器」權限）執行 → 把**當前頻道設為指定頻道**（起頭或更改），並張貼面板。此指定頻道同時是每日提醒的發送地點。
- **一般成員**執行 →
  - 在**指定頻道**：✅ 正常張貼面板
  - 在**其他頻道**：❌ 被擋下（「只能在管理員指定的頻道設置面板」）
  - 管理員**尚未指定**過任何頻道：⚠️ 提示先請管理員設置

> 日常請假、查看、取消都透過面板上的按鈕完成，不需指令，也不需要任何權限。

## 安裝步驟

1. **安裝依賴**
```bash
pip install -r requirements.txt
```

2. **設定 Discord Token**
   - 在專案根目錄建立 `.env` 檔案
   - 新增以下內容：
```
DISCORD_TOKEN=你的discord_bot_token
```

3. **運行機器人**
```bash
python main.py
```

4. **設置面板**
   - 由具「管理伺服器」權限的成員，在想放面板的頻道輸入 `!設置面板`
   - 此頻道會成為指定頻道（面板與每日提醒的所在地）
   - 之後所有人即可用面板上的按鈕操作

## Discord Bot 設定

1. 造訪 [Discord Developer Portal](https://discord.com/developers/applications)
2. 建立新應用，並在 "Bot" 分頁建立 bot
3. 複製 Token 貼到 `.env` 檔案
4. 在 "Bot" 分頁開啟 **Message Content Intent**（讀取訊息內容，指令前綴需要）
5. 在 "OAuth2" > "URL Generator" 選擇權限：
   - `bot`
   - `Send Messages`
   - `Embed Links`
   - `Attach Files`（月曆圖片需要）
   - `Read Message History`
6. 使用生成的 URL 邀請 bot 加入你的伺服器

## 使用方法

設置面板後，面板上有三個按鈕：

- **📅 請假申請** — 開啟日曆，依序選擇年 → 月 → 上/下半月 → 日期送出
- **📋 查看請假** — 顯示當月月曆圖（每格為當天人數），可按 ◀ 上月 / 下月 ▶ 切換，並用下拉選單查看某天有誰請假
- **✖️ 取消請假** — 列出你自己的請假，選一個即可取消

以上互動畫面皆為**僅本人可見**（ephemeral），不會洗版。

### 🔁 定期請假（每周X）

在「請假申請」畫面，**上半月／下半月**旁邊有第三顆 **🔁 定時請假**：

1. 按下後，上方的年、月兩個選單會**合併成一個星期選單**（日～六），日期選單隱藏
2. 選擇星期幾 → 立刻把**本月今天起**所有該星期幾排入請假
3. 按 上半月／下半月 即可回到一般模式

**規則說明：**
- 定期請假一律套用**今天所在的月份**（定時模式下沒有年/月選單）
- 每月 **1 號 09:00** 自動排入下個月，並在頻道發總結訊息
- 可同時設定**多個**星期幾（例：每周一 ＋ 每周三）
- 產生的假就是一般請假紀錄，可單獨取消某一天當作例外
- 不影響已過去的日子，同一人也不會重複請到同一天

**取消定期請假：** 在「取消請假」選單中，🔁 每周X 會**置頂**顯示。選擇後會詢問：

| 選項 | 效果 |
|------|------|
| **刪除當月假 ＋ 停止定期** | 清掉本月剩餘的假，之後不再自動請 |
| **刪除當月假 ＋ 保留定期** | 只清本月（當作這個月例外），下個月照常自動排入 |

## 數據儲存

所有資料存放於 `data/` 資料夾，屬於機器人執行資料，請勿手動編輯：

| 檔案 | 用途 |
|------|------|
| `data/leaves.json` | 現用請假（今天與未來） |
| `data/leaves_archive.json` | 歷史封存（已過去的請假，每天凌晨自動搬入） |
| `data/recurring.json` | 定期請假規則（誰設定了每周幾） |
| `data/config.json` | 設定，儲存管理員指定的面板頻道 ID（同時作為白名單與每日提醒發送地點） |

### 中文字型

月曆圖片需要中文字型。專案已內附 **Noto Sans CJK TC**（開源 OFL 授權，可自由散布）於 `fonts/` 資料夾，程式會**自動偵測**字型，順序為：

1. 專案 `fonts/` 內的任何 `.ttc` / `.otf` / `.ttf`（隨專案一起部署，跨平台最方便）
2. 常見 Linux 系統字型（`/usr/share/fonts/.../NotoSansCJK-Regular.ttc`）
3. Windows 微軟正黑體（`C:/Windows/Fonts/msjh.ttc`）

因此**部署到 Google Cloud 等 Linux 環境時，整包帶著 `fonts/` 即可，不需額外設定**。若想換字型，把字型檔放進 `fonts/` 即可自動採用。

> 註：微軟正黑體為專有字型，請勿放入公開 repo 散布；`fonts/` 內附的 Noto 字型才可自由散布。
